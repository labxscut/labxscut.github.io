Python Interface

DeepLB Python API

Current Python support is compatibility-first and wraps the existing shell pipeline behavior.

Package entrypoints

  • Package: deeplb
  • Public import: from deeplb import run_pipeline
  • CLI: python -m deeplb, optional deeplb command after install

Function reference

run_pipeline(script_args=None, *, dry_run=False, check=True, capture_output=False)

Thin wrapper for Scripts/DeepLB_pipeline.sh.

  • script_args: pass-through shell args
  • dry_run: append -d if missing
  • check: raise on non-zero shell exit
  • capture_output: capture stdout/stderr

Returns subprocess.CompletedProcess[str].

from deeplb import run_pipeline

result = run_pipeline(
    ["-r", "/path/to/DeepLB", "-t", "lihc", "-g", "TH", "-u", "part3", "-q", "0.1", "-k", "hyper", "-v", "1"],
    dry_run=True,
    check=False,
    capture_output=True,
)
print(result.returncode)
print(result.stdout)

CLI wrapper

python -m deeplb supports:

  • --dry-run to append -d
  • Trailing arguments forwarded to Scripts/DeepLB_pipeline.sh
python -m deeplb -h
python -m deeplb -- -r /path/to/DeepLB -t lihc -g TH -u part3 -q 0.1 -k hyper -v 1
python -m deeplb --dry-run -- -r /path/to/DeepLB -t lihc -g TH -u part3 -q 0.1 -k hyper -v 1
